複数サイト。

このトップページは、1 つの GitHub Pages repo の中に複数サイトを並べて置くための入口です。 各サイトはサブフォルダごとに分け、画像や CSS は共有アセットとして管理できます。

site-a を開く site-b を開く

docs/assets/ に共通ファイルを置き、各サイトから相対パスで参照できます。

2 個別サイトのサンプル。`site-a` と `site-b` を追加済みです。
1 Repo 公開は 1 回、ルーティングはサブフォルダで分離します。
Shared CSS や画像を共有すれば、追加サイトの管理が軽くなります。
Pages Ready 今の workflow のまま `docs/` 全体をそのまま公開できます。

おすすめ構成

1 repo で複数サイトを運用するときは、トップページと各サイトをこの形で分けると扱いやすいです。

Folder Layout
Root

docs/index.html

一覧ページ。各サイトへのリンクを並べる入口です。

  • `/site/` に相当するトップ URL
  • 全サイトの目次として使えます
Top page
Shared

docs/assets/

画像、CSS、JS などの共通ファイル置き場です。

  • 複数サイトから共通で参照可能
  • 変更箇所をまとめやすいです
Shared assets
Sites

docs/site-x/

個別サイトごとの HTML と専用画像を置く場所です。

  • URL はフォルダ名そのまま
  • 各サイトに `index.html` を置きます
Per site

増やし方

新しいサイトを足すときは、既存のサブフォルダをコピーして名前だけ変えるのがいちばん早いです。

1. サブフォルダを追加

例: docs/site-c/index.html を作ると /site/site-c/ で公開できます。

2. 相対パスで参照

画像や CSS は ../assets/ や ./assets/ を使って相対パスで読みます。

このページの本文はすべてプレースホルダーです。必要になったら自由に差し替えて使えます。
